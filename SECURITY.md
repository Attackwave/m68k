# Security Policy

## Supported Versions

Only the latest released version is supported with security fixes.

## Reporting a Vulnerability

If you discover a security vulnerability in this project (for example, a
crash, memory-safety issue, or panic reachable from untrusted input such as
a malformed floppy image or object file), please report it privately rather
than opening a public issue.

Use GitHub's [private vulnerability reporting](https://github.com/Attackwave/m68k/security/advisories/new)
feature for this repository. This lets us discuss and fix the issue before
it's publicly disclosed.

Please include:
- A description of the issue and its potential impact.
- Steps to reproduce (ideally a minimal input file or byte sequence).
- Which crate(s) and code path are affected, if known.

We'll acknowledge reports as soon as we can and aim to release a fix before
any public disclosure.
